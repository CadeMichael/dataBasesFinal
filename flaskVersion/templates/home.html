Technologies and FrameWorks

Flask
  • we decided on flask because of it's versatility and minimalism. There were several other "better" frameworks that are a little faster and more "batteries inlcuded." But flask did exactly what we needed and kept everything simple.
  • one of the main considerations was python. It's integration with databases and simplicity were one of the reasons we went with flask. Although JS isn't terrible I feel that python makes things a bit simplier, allowing us to focus on the database aspect instead of the programming aspect.
sqlite3
  • much like python sqlite keeps everything simple. Although not ideal for production sqlite is perfect for prototyping. It is local which means no fussing with servers or logging in. Additionally it is easily viewed and alterted by GUI programs, making checking the database easy. The deciding factor was its tight integration with python as python has a native module for sqlite.
